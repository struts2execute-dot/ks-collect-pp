# Build.js 域名验证绕过修改指南

## 概述

本文档记录了针对 Pragmatic Play 游戏 build.js 文件的修改方法，用于绕过域名验证和时间戳检查逻辑。

---

## 修改类型总览

| 类型 | 数量 | 说明 |
|------|------|------|
| 域名硬编码 | 2处 | 将动态获取的域名替换为固定字符串 |
| 错误返回修改 | 6-8处 | 将验证失败的返回值改为 `false` |
| 时间戳检查 | 2处 | 禁用时间戳过期检查 |

## 搜索模式

### 1. 域名硬编码（2处）

**搜索模式：**
```
grep -n "win\[.*\]\[.*\]()" build.js
```

**典型匹配：**
```javascript
var _0xxxxx = win[_0xxxxx(0xXXX)][_0xxxxx(0xXXX)]()
```

**修改方式：**
```javascript
// 原代码
var _0x11ad76 = win[_0x468cee(0xd7)][_0x468cee(0x18e)]()

// 修改为
var _0x11ad76 = "demogamesfree.pragmaticplay.net"
```

---

### 2. 错误返回修改（6-8处）

**搜索模式：**
```
grep -n "return.*|= !!\[\]" build.js
```

**典型匹配：**
```javascript
return _0xXXXXXX[_0xXXXXXX(0xXXX) + 'XX'] |= !![];
```

**修改方式：**
```javascript
// 原代码
return _0x5b1174[_0x468cee(0xe2) + 'de'] |= !![];

// 修改为
return false;
```

**注意事项：**
- 有些文件包含三元表达式形式，需要整体替换为 `false`
- 死代码分支中的返回也建议一并修改

---

### 3. 时间戳检查（2处）

**搜索模式：**
```
grep -n "|= Math\[.*\](.*) > 0xe10" build.js
```

**典型匹配：**
```javascript
_0xXXXXXX[_0xXXXXXX(0xXXX) + 'XX'] |= Math[_0xXXXXXX(0xXXX)](_0xXXXXXX) > 0xe10 * 0x30;
```

**修改方式：**
```javascript
// 原代码
_0x5b1174[_0x845130(0x140) + 'a9'] |= Math[_0x845130(0xe1)](_0x290de2) > 0xe10 * 0x30;

// 修改为
_0x5b1174[_0x845130(0x140) + 'a9'] = false;
```

## 自动化修改脚本

```python
#!/usr/bin/env python3
import sys
import re

def patch_build_js(filepath):
    with open(filepath, 'r') as f:
        content = f.read()
    
    original_content = content
    
    # 1. 域名硬编码替换
    # 匹配: var _0xXXXXXX = win[_0xXXXXXX(0xXXX)][_0xXXXXXX(0xXXX)]()
    pattern1 = r'(var _0x[a-f0-9]+) = win\[_0x[a-f0-9]+\(0x[a-f0-9]+\)\]\[_0x[a-f0-9]+\(0x[a-f0-9]+\)\]\(\)'
    content = re.sub(pattern1, r'\1 = "demogamesfree.pragmaticplay.net"', content)
    
    # 2. 错误返回替换
    # 匹配: return _0xXXXXXX[_0xXXXXXX(0xXXX) + 'XX'] |= !![];
    pattern2 = r"return _0x[a-f0-9]+\[_0x[a-f0-9]+\(0x[a-f0-9]+\) \+ '[a-f0-9]+'\] \|= !!\[\];"
    content = re.sub(pattern2, 'return false;', content)
    
    # 3. 时间戳替换
    # 匹配: |= Math[...](...) > 0xe10 * 0x30;
    pattern3 = r"(_0x[a-f0-9]+\[_0x[a-f0-9]+\(0x[a-f0-9]+\) \+ '[a-f0-9]+'\]) \|= Math\[_0x[a-f0-9]+\(0x[a-f0-9]+\)\]\(_0x[a-f0-9]+\) > 0xe10 \* 0x30;"
    content = re.sub(pattern3, r'\1 = false;', content)
    
    with open(filepath, 'w') as f:
        f.write(content)
    
    print(f"Modified: {filepath}")
    print(f"Original size: {len(original_content)}")
    print(f"New size: {len(content)}")

if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("Usage: python patch_build.py <build.js>")
        sys.exit(1)
    patch_build_js(sys.argv[1])
```

## 注意事项

1. **混淆变量名**：每个文件的变量名都不同（如 `_0x468cee`, `_0x4400db` 等），但代码结构相同
2. **行号差异**：不同文件的行号可能略有差异，应以搜索模式为准
3. **三元表达式**：部分文件包含三元表达式形式的验证，需要整体替换
4. **死代码分支**：建议将所有匹配的代码都进行修改，包括永远不会执行的分支
